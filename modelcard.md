# ModelCard.md# Model Card## Interpretable Longitudinal Multi-Omics Prediction of Incident Type 2 Diabetes## Model SummaryThis project contains a set of **synthetic benchmark models** for predicting **incident type 2 diabetes** from simulated longitudinal clinical, genetic, transcript-like, proteomic, and metabolomic features. The primary predictive model is an **L1-penalised logistic regression** trained on nested feature sets ranging from clinical-only to integrated multi-omics plus trajectory summaries.The best-performing configuration in this benchmark is the **Clinical+MultiOmics+Trajectory** model, which combines standard clinical variables, a synthetic polygenic risk score, transcript-like features, protein-like features, metabolite-like features, and longitudinal slope-derived summaries.## Model Details### Model Type- Binary classification model- Primary estimator: L1-penalised logistic regression- Training framework: scikit-learn- Evaluation: five-fold stratified cross-validation and separate holdout interpretation### Input Modalities- Clinical variables- Behavioural variables- Polygenic risk score- Transcript-like omics features- Protein-like omics features- Metabolite-like omics features- Longitudinal slope features- Simulated subgroup indicators for robustness analysis### Output- Predicted probability of incident type 2 diabetes- Binary classification at a chosen threshold- Risk decile assignment for stratification analyses## Intended Use### Appropriate Uses- Demonstrating a complete disease-prediction workflow- Teaching interpretable machine learning for biomedicine- Benchmarking notebook structure for longitudinal multi-omics studies- Prototyping disease prediction pipelines before applying to real cohorts- Portfolio, concept note, or manuscript development### Inappropriate Uses- Clinical diagnosis- Medical decision support- Risk counselling for individuals- Regulatory submission- Real-world deployment- Any inference about real populations or ancestry groups## Training DataThe model was trained on **synthetic data only**. No real human subjects, patient records, or identifiable health records were used. The simulation incorporates broad directionality inspired by published literature on T2D risk biomarkers but does not reproduce any specific real-world cohort.### Synthetic Feature DesignThe data generation process includes:- age, sex, BMI, smoking, activity, family history,- fasting glucose, HbA1c, HDL, triglycerides, systolic blood pressure,- a synthetic polygenic risk score,- latent insulin resistance, inflammation, and beta-cell stress axes,- transcript-like, protein-like, and metabolite-like features,- repeated measurements at years 0, 2, and 5,- event generation over approximately 5.5 to 8 years.## PerformancePerformance should be interpreted only within the synthetic benchmark environment.### Reported Metrics- AUROC- AUPRC- Brier score- Accuracy- F1 score- Expected calibration error- Risk-decile enrichment- Subgroup AUROC and calibration-related summaries### Expected PatternWithin the synthetic setup:- clinical-only models provide a strong baseline,- PRS adds modest value,- proteomics and metabolomics add incremental discrimination,- combined multi-omics plus trajectories provide the best performance,- calibration may vary with model complexity,- subgroup performance may differ across simulated strata.## InterpretabilityThis benchmark places strong emphasis on interpretability.Interpretation tools include:- sparse logistic regression coefficients,- permutation importance on the holdout set,- bootstrap feature-selection stability,- subgroup performance analysis,- risk-decile event enrichment,- Kaplan-Meier-like temporal stratification.These methods are included to support transparent model inspection rather than black-box prediction alone.## Ethical ConsiderationsThis model is synthetic and should not be mistaken for a clinically valid predictor. Important ethical considerations include:- It should not be used for patient care.- Subgroup labels are simulated and must not be interpreted as real ancestry effects.- Good overall performance does not guarantee equitable subgroup performance.- Biological plausibility in a simulation does not establish causal truth.- Translational claims require validation in real, diverse, ethically governed cohorts.## Limitations- Entirely synthetic dataset- Simulated effect sizes and correlations- Simplified longitudinal sampling- No medication exposure modelling- No dropout mechanism beyond simple censoring- No external validation dataset- No true causal inference- No calibration recalibration step- Limited model class diversity## MaintenanceThis model card describes the current synthetic proof-of-concept version. If the notebook is later adapted to real cohort data, this card should be updated to reflect:- real training data provenance,- population coverage,- inclusion and exclusion criteria,- data preprocessing,- external validation,- fairness analyses,- calibration correction,- deployment constraints.## Contact and ProvenanceThis model card accompanies a synthetic notebook-based proof of concept for interpretable longitudinal multi-omics T2D prediction. It is intended as a transparent record of design goals, limitations, and appropriate use boundaries.
